## APIs

S-6 exposes Python module APIs (no HTTP server/boundary). All outbound LLM calls are orchestrated via S-3 (`complete()`).

### `propagate_edits()`

**Module:** `src.core.propagate`

**Input Payload Schema:**
| Field            | Type                 | Nullable | Constraints                                                                 |
|------------------|----------------------|----------|-----------------------------------------------------------------------------|
| `edited_title`   | `str`                | No       | Length 1-255 chars. Used to build regex `\[\[{escaped_title}\]\]`.         |
| `edited_content` | `str`                | No       | Length 1-750,000 chars (bounded by S-3 inputs).                            |
| `pending`        | `list[PageProposal]` | No       | Min length 0. Bounded roughly by A-10 (~15 proposals constraint).           |
| `provider`       | `str`                | No       | Exact enum: `"gemini"` \| `"lmstudio"`.                                     |
| `config`         | `Config`             | No       | Bounded object originating from S-1 `config.json`.                         |
| `env`            | `dict`               | No       | Must contain runtime API keys if `provider == "gemini"`.                    |
| `source_text`    | `str`                | No       | Original extracted textual source. Length 1-750,000 chars.                 |
| `source_meta`    | `dict`               | No       | Dict must contain key `"source"`, validating extraction context metadata.   |

**Returns:** `list[PageProposal]`
- Order is rigorously preserved per input list.
- Unaffected dependents are identical to their input state.
- Dependent proposals are fully updated with re-evaluated content/conflict schemas natively generated.

**Failure Semantics:** 
- If `complete()` faults or triggers `SystemExit` for one page, the function captures the error, issues a stdout warning, retains original unmutated `PageProposal` inside the `pending` queue, and **continues evaluation on the next dependent**.

### `find_dependents()`

**Module:** `src.core.propagate`

**Input Payload Schema:**
| Field          | Type                 | Nullable | Constraints                                                |
|----------------|----------------------|----------|------------------------------------------------------------|
| `edited_title` | `str`                | No       | Length 1-255 chars.                                       |
| `proposals`    | `list[PageProposal]` | No       | Search targets. Min length 0.                              |

**Returns:** `list[int]` 
- Ascending indices mapping to matched items in `proposals`.

**Failure Semantics:** 
- Throws `ValueError` if `edited_title` is null or zero-length.

### `_re_evaluate_page()` (Internal Bounder)

**Module:** `src.core.propagate`

**Input Payload Schema:**
| Parameter      | Type  | Description                                 |
|----------------|-------|---------------------------------------------|
| Same bounds as | Above | Wrapped variables supplied via caller.      |
| `proposal`     | `PageProposal` | Strict definition below.           |

**Returns:** `PageProposal` (re-parsed structure with updated content/conflict blocks).

**Failure Semantics:** Propagates `SystemExit` generated by S-3 `complete()`.

## Data Structures

### `PageProposal`

*Originates in S-4. Validated across S-6 boundaries to establish reliable mutations.*

| Field           | Type              | Null | Length Constraints      | Notes                                                              |
|-----------------|-------------------|------|-------------------------|--------------------------------------------------------------------|
| `title`         | `str`             | No   | 1 - 255 chars           | Compared against `edited_title` for linking via case-agnostic regex|
| `slug`          | `str`             | No   | 1 - 255 chars           | Persisted across evaluation unchanged.                             |
| `category`      | `str`             | No   | 1 - 64 chars            | Usually one of pre-defined buckets, preserved.                    |
| `content`       | `str`             | No   | 1 - 750,000 chars       | Scanned for `[[wikilinks]]`. Supplied to LLM for evaluating delta. |
| `action`        | `str`             | No   | `"create"`\|`"update"`| Preserved structure.                                               |
| `conflicts`     | `list[Conflict]`  | No   | Max size boundless      | Empty sequence if `action == "create"`.                            |
| `existing_path` | `str`             | Yes  | 0 - 1024 chars          | Required string path if `action == "update"`. None if `create`.    |

### `Conflict`

| Field             | Type   | Null | Length Constraints      | Notes                                                  |
|-------------------|--------|------|-------------------------|--------------------------------------------------------|
| `description`     | `str`  | No   | 1 - 1024 chars          | S-6 parser retrieves from `⚠️ CONFLICT:` LLM marker. |

### Wikilink Regex Pattern
- **Format**: `\[\[{escaped_title}\]\]`
- **Flags**: `re.IGNORECASE`
- **Dependency Map**: Matches any literal occurrence disregarding case inside `PageProposal.content`.

## Boundary Limits

| Metric                 | Constraint                                         | Rationale                                                         |
|------------------------|----------------------------------------------------|-------------------------------------------------------------------|
| **Rate Limit**         | Dictated exclusively by Provider Limit (S-3)       | e.g., 15 RPM for Gemini free-tier. Fallbacks disabled for MVP.    |
| **Latency/Timeouts**   | Sequential processing constraint: 5s - 120s / page | Because logic blocks interactive loop, worst-case user waits 120s.|
| **Max Payload Size**   | Content Chunk Upper Bound: ~750,000 tokens         | Maximum content load generated via source string (Gemini Context).|
| **In-Memory Scale**    | Number of edits per pending size: O(15 pages)      | Bounded scale limits memory inflation.                            |

## Tests

**Execution File**: `tests/unit/test_s6_propagation.py`

### Happy Path
1. **Single match**: Proposal list of length 3, middle proposal contains `[[Target]]`. Call to `propagate_edits("Target", ...)` re-evaluates strictly index 1 and returns cloned 3-item list.
2. **Case Insensitivity**: Title `"Attention Mechanism"` correctly recognizes `[[attention mechanism]]` in proposals.
3. **Regex Accuracy**: Assure bounding validation handles characters explicitly without partial substring matches (i.e., avoids matching `[[Attention Mechanism Overview]]` when edit is merely `"Attention Mechanism"`).

### Edge Cases
1. **Empty Pending Stack**: Call `propagate_edits` with `pending=[]`. Ensure returns immediately without invoking LLM or failing.
2. **Absent Dependents**: Call `propagate_edits` where `find_dependents` yields `[]`. Returns input identical list.
3. **Escaped Characters in Titles**: Provide an edited_title matching `C++ (Language)`. Confirm valid regex formation detects `[[C++ (Language)]]` reliably.

### Failure Paths
1. **Missing Identifiers**: Execute `propagate_edits` with `edited_title=""`. Expect fast-fail raising `ValueError`.
2. **LLM `SystemExit` Propagated**: Mock `complete()` generating `SystemExit` inside `_re_evaluate_page()`. Assert that the loop consumes `SystemExit`, writes an explicit error warning out to the log/console, yet strictly leaves the previous iteration `PageProposal` preserved untouched inside the list structure.
3. **All Re-evaluations Fail**: Batch of two explicitly matching dependents yield LLM timeouts side-by-side. Evaluates that both proposals fall back accurately returning full pristine initial state list without terminating the tool.
