## APIs

S-8.5 acts as the top-level procedural orchestrator and exposes a core python routing interface to the CLI wrapper. It relies entirely on existing data models. 

### `execute_ingestion(source: str) -> None`

| Aspect | Spec |
|--------|------|
| Module | `src.core.ingest` |
| Input | `source` - A string representing an absolute/relative path or accessible URL. |
| Output | `None`. Operations perform structural state mutation (Vault file updates) entirely through nested `commit_approved_batch` integrations. Output logging tracks via standard `print()` mechanisms routing active status indicators to the CLI. |
| Raises | None directly natively; explicitly bubbles unhandled `SystemExit` and `KeyboardInterrupt` for terminal management. |
| Behavior | 1. Maps environment/configs via `load_env()` and `load_config()`.<br>2. Captures Provider instances structurally via `select_provider()`.<br>3. Dispatches payload extracts to `extract_source()`.<br>4. Orchestrates `generate_batch()`.<br>5. Binds returned payloads inside `$EDITOR` integrations using `review_batch()`.<br>6. Routes edit interactions downwards to `propagate_edits()`.<br>7. Closes loop saving successful edits utilizing `commit_approved_batch()`. |
| Idempotent | No. Success mutates structural vault state natively executing commits. |
